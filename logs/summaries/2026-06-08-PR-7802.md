## Root Cause

Both PRs originated from the same base region in `general-mirroring` (Rob2309). PR #7003 landed in the build first, establishing the classmethod pattern. PR #7802 was independently rebased onto v0.8.0 with its own (equivalent but differently-shaped) mirror-sync implementation.

Key ancestor commit: **`442d716dd6`** ("Door materials are now synced to mirrored types") — this is the original source of both the `material_target_element` fix in `update_door_modifier_representation` AND the `_finish_one` classmethod structure. After PR #7003's rebase, `c654e58ca3` became the tip containing the final classmethod shape.

## Fix Strategy: `git merge -s ours c654e58ca3`

**Why ancestry-merge works here:**

With merge base = v0.8.0 (`93c6350e0f`):
- All three conflict files had additions from *both* sides → genuine 3-way conflict.

After `git merge -s ours c654e58ca3` on PR #7802's branch:
- Merge base advances to `c654e58ca3` (PR #7003's tip).
- **door.py:** Build's `FinishEditingDoor` is *unchanged* from `c654e58ca3`; PR #7802 changed it → auto-merge takes PR #7802's instance-method version (no conflict).
- **opening.py:** Build's blank-line area is *unchanged* from `c654e58ca3`; PR #7802 has 3 blank lines → auto-merge takes PR #7802's version (no conflict).
- **workspace.py:** Build's `hotkey_C_F` is *unchanged* from `c654e58ca3`; PR #7802 adds `keep_original=` arg → auto-merge takes PR #7802's improvement (no conflict).

**Result in merged build:**
- `FinishEditingDoor` uses PR #7802's instance-method approach (functionally equivalent to PR #7003's classmethod approach; both sync mirrored door params).
- `hotkey_C_F` gets the `keep_original=getattr(self, "shift", False)` improvement from PR #7802.
- All other changes (duplicate opening logic in `FilledOpeningGenerator`, `AddDoor` material loading from PR #7653, extend-to-cursor from PR #7648, clone-opening fixes from PR #7766) merge cleanly in non-overlapping regions.
