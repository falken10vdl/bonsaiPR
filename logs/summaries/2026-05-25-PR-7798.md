When git then tries to merge ManualDrawingReference:

| | workspace.py around the `add_layout_hotkey_operator` block |

| **Merge base** (`913c9c42ba`) | `_DIMENSION_TYPES` block only — no `is_manual_reference` |
| **Build HEAD** (after PR #7965) | `is_manual_reference` **before** `_DIMENSION_TYPES` |
| **ManualDrawingReference** (`66fcad84e3`) | `is_manual_reference` **after** `_DIMENSION_TYPES` |

Both sides added `is_manual_reference` relative to the base but at **different positions**
→ git's 3-way merge cannot auto-resolve → conflict.

## Why the Blocks Are Mutually Exclusive

- `is_manual_reference` only shows when `object_type in ("ELEVATION", "SECTION")`
- `force_perpendicular_to_face` only shows when `object_type in _DIMENSION_TYPES`
  (DIMENSION, RADIUS, DIAMETER, ANGLE, PLAN_LEVEL, SECTION_LEVEL)

These sets are disjoint, so the display order has no runtime effect — it's purely a UI
layout ordering preference.

## Fix Strategy: Content Fix (new commit)

The ManualDrawingReference branch needed a content change — moving `is_manual_reference`
from after `_DIMENSION_TYPES` to before it. This also matches the **original PR's intent**:
the original PR tip (`565414cf51`) placed `is_manual_reference` before anything related to
`_DIMENSION_TYPES` (which didn't exist in the base at that time).

A new commit was added to ManualDrawingReference:
[`3409c656ca`](https://github.com/IfcOpenShell/IfcOpenShell/commit/3409c656ca)

```python
        add_layout_hotkey_operator(cls.layout, "Add", "S_A", "Create a new annotation")

        if object_type in ("ELEVATION", "SECTION"):   # ← now BEFORE _DIMENSION_TYPES
            row = cls.layout.row(align=True)
            row.prop(cls.props, "is_manual_reference")

        _DIMENSION_TYPES = {"DIMENSION", ...}
        if object_type in _DIMENSION_TYPES:
            row = cls.layout.row(align=True)
            row.prop(cls.props, "force_perpendicular_to_face")
```

### 3-way merge mechanics after the fix

After the fix, when the build (which has `is_manual_reference` before `_DIMENSION_TYPES`
from PR #7965) tries to merge ManualDrawingReference:

- **Base** (`913c9c42ba`): no `is_manual_reference`
- **Build**: `is_manual_reference` before `_DIMENSION_TYPES` (via PR #7965)
- **ManualDrawingReference** (fixed): `is_manual_reference` before `_DIMENSION_TYPES`

Both sides made the **same addition** at the **same position** relative to the base.
Git's 3-way merge keeps it once — clean merge.
