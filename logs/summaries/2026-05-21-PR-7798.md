## Why They Conflict

Both PRs independently chose the same sequential IFC entity IDs and Python property insertion points, because both were branched from v0.8.0 before the other's changes existed there.

### File 1: `src/bonsai/bonsai/bim/data/pset/Psets_BBIM_Annotation.ifc`

The v0.8.0 base has IFC entities up through `#33`. Both PRs appended new entries starting at `#34`:

- **PR #8083** added `#34=Separator`, `#35=SuppressZeroFeet`, `#36=IsOrdinate`, `#37=Anchors`, `#38=MeasureAxis`, `#39=ForcePerpendicularToFace`, `#40=LinePosition` for the `BBIM_Dimension` pset.
- **PR #7798** added `#34=IsManualDrawingReference`, `#35=IsDocumentReference` for the `EPset_Annotation` pset.

Both sides also updated the header entry (`#1`) to reference their new IDs: `(#4,#33,...,#34,#35)`. Git's 3-way merge sees both sides modifying the same region (after `#33`) relative to the LCA (v0.8.0), causing a conflict.

### File 2: `src/bonsai/bonsai/bim/module/drawing/prop.py`

Both PRs appended a new `bpy.props.*Property` to `BIMAnnotationProperties` immediately after `type_name`:

- **PR #8083** added `force_perpendicular_to_face` (BoolProperty) and `line_position` (FloatProperty).
- **PR #7798** added `is_manual_reference` (BoolProperty).

### File 3: `src/bonsai/bonsai/bim/module/drawing/workspace.py` (two conflict regions)

**Region 1** — UI row after "Add annotation" hotkey operator:
- **PR #8083** added a dimension-type check showing `force_perpendicular_to_face`.
- **PR #7798** added an elevation/section check showing `is_manual_reference`.

**Region 2** — `hotkey_S_A` method:
- **PR #8083** restructured it to dispatch parametric-dimension type annotations to `draw_parametric_dimension`, falling back to `add_annotation()`.
- **PR #7798** changed `add_annotation()` → `add_annotation("INVOKE_DEFAULT")` for modal invoke behavior.

## Fix Strategy: Ancestry-Merge (Option A)

**Why ancestry-merge, not rebase:**  
PR #7798's content intent is fully independent of #8083. The manual-drawing-reference feature (IsManualDrawingReference flag, manual reference tag UI) does not logically depend on parametric dimensions. The conflict arises purely from git's 3-way merge seeing parallel insertions at the same location relative to their shared ancestor (v0.8.0), not from one PR's code needing to adapt to the other's API changes.

**How the LCA shift resolves it:**  
By merging #8083's branch head (`913c9c42ba`) into `ManualDrawingReference`, that commit becomes an ancestor of #7798. When the build (which also has `913c9c42ba` as an ancestor via the `7a474cd7f4` merge commit) then tries to merge #7798:

- The merge-base (LCA) between the build tip and `ManualDrawingReference` shifts from v0.8.0 to `913c9c42ba` (the #8083 head).
- From that LCA, the build side made no further changes to the three conflicting files (confirmed by inspecting build commits `7a474cd7f4..97b83c43d0`).
- From that LCA, `ManualDrawingReference` contributes only the annotation-specific additions (renumbered IDs, `is_manual_reference` property, elevation/section UI row, and `"INVOKE_DEFAULT"` argument).
- Git's 3-way merge sees no contest on either side — clean merge.

## Conflict Resolution Details

### `Psets_BBIM_Annotation.ifc`

Kept #8083's IDs (#34–#40) unchanged (they are referenced throughout `BBIM_Dimension`). Renumbered #7798's entries:

- `#41 = IsManualDrawingReference` (was #34 in #7798)
- `#42 = IsDocumentReference` (was #35 in #7798)

Updated the `EPset_Annotation` header reference list from `(...,#34,#35)` to `(...,#41,#42)`.

### `prop.py`

Kept all three properties, placing #8083's `force_perpendicular_to_face` and `line_position` first, then #7798's `is_manual_reference`. Order within the class is cosmetic.

### `workspace.py` — UI rows

Kept both conditional blocks sequentially: dimension-types check first (from #8083), elevation/section check second (from #7798). They are mutually exclusive by `object_type`, so order has no runtime effect.

### `workspace.py` — `hotkey_S_A`

Used #8083's dispatch structure (parametric dimension routing) with #7798's `"INVOKE_DEFAULT"` argument applied to the `add_annotation` call:

```python
def hotkey_S_A(self):
    props = tool.Drawing.get_annotation_props()
    if props.object_type in self._PARAMETRIC_DIMENSION_TYPES:
        if bpy.ops.bim.draw_parametric_dimension.poll():
            bpy.ops.bim.draw_parametric_dimension("INVOKE_DEFAULT")
    elif bpy.ops.bim.add_annotation.poll():
        bpy.ops.bim.add_annotation("INVOKE_DEFAULT")
```

This preserves both PRs' intent: #8083's parametric-dimension operator routing, and #7798's modal invocation of `add_annotation`.
